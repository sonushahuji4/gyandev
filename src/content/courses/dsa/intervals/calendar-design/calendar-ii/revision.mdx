<RevisionCard numeral="i" title="Problem" subtitle="LeetCode #731 — one-liner.">
- Allow **double** bookings, but reject anything that would create a **triple** booking.
</RevisionCard>

<RevisionCard numeral="ii" title="Key insight" subtitle="Track the double-booked regions.">
- Keep two lists: `bookings` (all accepted events) and `overlaps` (the regions where two events already coincide).
- A new booking is rejected iff it overlaps any `overlaps` region (that would be three-deep).
- Otherwise, record the intersections with each existing booking into `overlaps`, then append the booking itself.
</RevisionCard>

<RevisionCard numeral="iii" title="Solution template" subtitle="Two scans per call.">
```python
# reject if intersects any existing double-booking
for os, oe in overlaps:
    if start < oe and end > os: return False
# record new double-bookings
for bs, be in bookings:
    if start < be and end > bs:
        overlaps.append((max(start, bs), min(end, be)))
bookings.append((start, end)); return True
```
</RevisionCard>

<RevisionCard numeral="iv" title="Complexity" subtitle="Per-call cost.">
- **Time `O(n)` per call** — two linear scans over the stored lists.
- **Space `O(n)`** — bookings + overlaps grow together.
</RevisionCard>

<RevisionCard numeral="v" title="Edge cases" subtitle="Two to remember.">
- Overlapping with an existing double-booking → reject (would be triple).
- An accepted booking that overlaps several existing ones records a separate double-booked region per existing booking.
</RevisionCard>

<Callout type="mnemonic" title="Mnemonic">
  **Two · Layers · Tracked** — bookings *and* overlaps, checked in that order.
</Callout>
