## What a closure actually is

A closure is a function plus the lexical environment it was defined in. When the inner
function is invoked later, it still sees the variables that were in scope when it was
created — even if the outer function has already returned.

```js
function counter() {
  let n = 0;
  return () => ++n;
}

const tick = counter();
tick(); // 1
tick(); // 2
```

The returned arrow function *closes over* `n`. `n` lives as long as `tick` does.

## Why this matters

Closures give you three things for free:

1. **Private state** — nothing outside `counter` can read or write `n`.
2. **Factory functions** — configure once, reuse the returned function.
3. **Callbacks with memory** — event handlers, timers, and promise `.then` callbacks
   remember the data that was relevant when they were set up.

## The foot-gun

A closure keeps every variable in its enclosing scope alive, not just the ones it obviously
uses. If you capture a huge object by accident, it cannot be garbage collected until the
closure itself is released.

Rule of thumb: if a closure is long-lived (an event listener, a global callback),
double-check what it captures.
