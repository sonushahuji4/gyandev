# Accessibility runbook

Owner plan: [shared/accessibility.md](../.claude/plans/shared/accessibility.md). This document is the operational companion — what CI enforces, what to walk through by hand, which primitive to reach for, and how to triage the failures CI flags.

## Overview

GyanDev targets **WCAG 2.2 Level AA** across every page type. Three of the new 2.2 criteria matter most for this codebase:

- **2.4.11 Focus Not Obscured (Minimum)** — the sticky top nav cannot hide the focused element. Enforced by `scroll-padding-top` on `<html>` + native `<dialog>` for modals.
- **2.5.7 Dragging Movements** — swipe-to-dismiss on drawer / bottom sheet always has an X button + backdrop click as non-drag fallbacks.
- **2.5.8 Target Size (Minimum)** — every interactive element is ≥ 24×24 CSS px. Phone-layout targets are raised to ≥ 44×44 via the `TouchTarget` wrapper.

Level AA contrast floor: **4.5:1** for body text, **3:1** for large text / non-text UI. Both themes must pass — `scripts/check-contrast.mjs` asserts this at build time against `scripts/contrast-pairs.json`.

## Automated checks in CI

All of these run in `.github/workflows/ci.yml` on every PR. Merge is blocked if any fails.

| Check | Command | What it catches |
|---|---|---|
| Token-pair contrast | `npm run check:contrast` | Any `--color-*` pair listed in `scripts/contrast-pairs.json` that drops below its required ratio in either theme. |
| axe-core scan | `npx @axe-core/cli http://127.0.0.1:4321 --exit` | Rule-based DOM audit against rendered pages; fails on any violation (serious + critical blocked). |
| Pa11y scan | `npx pa11y-ci --config .pa11yci.json` | Second-opinion WCAG2AA runner across the five Phase-1 URLs listed in `.pa11yci.json`. |
| Lighthouse a11y | `npx lhci autorun --config=.lighthouserc.cjs` | Asserts `categories:accessibility: 0.95` minimum; heading structure, ARIA, contrast, label-for-id. |

axe + Pa11y probe a running `astro preview` server (`--host 127.0.0.1 --port 4321`). Lighthouse uses `staticDistDir: './dist'` with an internal server, so its URL list points at `http://localhost:8080/*.html` (see `.lighthouserc.cjs`).

Noise overrides documented in `.lighthouserc.cjs`: `label-content-name-mismatch` and `network-dependency-tree-insight` are downgraded to `warn` with a comment explaining why — do not drop them silently.

## Manual testing protocol

Automated gates cover ~60% of WCAG issues. The rest comes from doing the flow yourself.

### Keyboard-only walkthrough

Run once per PR on any page the PR touches, and once fully across the site per release.

1. Load the page, hit `Tab` from the URL bar.
2. First stop must be the **SkipLink** ("Skip to main content") — visible with focus ring. Enter jumps to `<main id="main">`.
3. Continue tabbing through top nav → breadcrumbs → main content → footer. Every interactive element shows a visible 2px focus outline. Nothing is reachable only by mouse.
4. On the chapter page, test `←` / `→` to navigate to adjacent chapters, `Mod+K` and `/` to open search, `Esc` to close it.
5. In the mobile drawer + search modal, `Tab` must stay trapped inside; `Esc` closes and returns focus to the trigger.
6. At 200% browser zoom, no horizontal scroll appears and no content is clipped (WCAG 1.4.10 Reflow).

### Screen reader

- **VoiceOver (macOS)** — `Cmd+F5` to start. Walk home + one chapter per release. Verify: `<h1>` read after `astro:page-load`, tab-bar announces "tab 2 of 3 selected", theme toggle announces "Dark mode on" via `LiveRegion`.
- **NVDA (Windows)** — same pages, before every release. Watch for heading-level skips and unlabelled icon buttons that VoiceOver's heuristics forgive but NVDA does not.

Record the walkthrough verdict (pass / what failed) on the release issue. The VoiceOver run of home + chapter is a launch gate (§5.4).

## Per-page a11y verification checklist

Every new page/template PR must re-verify the row that matches it. For a multi-page PR, run the row for every template touched.

| Page type | Landmark check | Keyboard check | Screen reader check |
|---|---|---|---|
| Home (`/`) | `<header>`, `<main id="main">`, `<footer>` exist once each | SkipLink → nav → hero CTA → featured strip → recent list → footer | H1 read after swap; Continue-Reading card announces progress % |
| All-courses (`/courses`) | Grid is a `<ul>` of `<article>`s; progress widget has `role="progressbar"` with `aria-valuenow/min/max` | Arrow keys move focus across cards; Enter follows link | "X of Y chapters complete" read from the progress widget |
| Course overview (`/courses/[course]`) | Single `<h1>` = course title; objectives + prereqs in `<ul>` | Chapter list is tab-reachable; Resume CTA is focusable before chapter list | Chapter list reads as ordered (lexical order from `order` field) |
| Chapter Full Notes (`/courses/[course]/[chapter]`) | `role="tablist"` on tab bar; active tab has `aria-selected="true"` + `aria-current="page"`; single `<h1>` (layout-owned) | `←`/`→` moves tab focus; Enter navigates; TOC is keyboard-reachable; Mark-as-read announces via LiveRegion | `<h1>` read; figure descriptions read for any `<FigureDescribed>` |
| Chapter Revision / Flow | Same tab-bar ARIA; canonical back to Full Notes; `noindex` does not affect a11y | Same keybindings work (view persists across tab swap) | Mermaid SVG on Flow is wrapped in `FigureDescribed` with a prose description |
| 404 | Single `<h1>` = "Page not found"; search input is first focusable after SkipLink | Tab reaches search, course list, GitHub issue link | Announces page title after redirect |
| About / Privacy / Terms | Single `<h1>`; prose headings descend correctly (h2 → h3, no skips) | Skip-link works; in-page anchors are keyboard-reachable | Read as prose without ARIA noise |
| Search modal (`/search`) | `role="dialog"` with `aria-labelledby`; Pagefind results in a live region | `Esc` closes and returns focus to trigger; `↑` / `↓` moves through results | Result count announced when results arrive |

## Common issues + fixes

### Color contrast failures

`check:contrast` fails with `pair X/Y ratio 3.8 < 4.5 (dark)`. Fixes in order of preference:

1. **Darken the foreground token** in `global.css` — `--color-text-muted` is the usual offender in dark mode.
2. **Lighten the background** only if it's a surface token used in one place.
3. Never lower the required minimum in `contrast-pairs.json`. If a UI element is explicitly "large text" (18pt+ or 14pt bold), add it as a new pair with `min: 3.0` rather than relaxing body text.

Re-run with `npm run check:contrast` locally before pushing; the script prints per-theme ratios so you can see both states in one go.

### Focus traps (wrong trap, not missing trap)

Symptom: `Tab` leaks out of an open modal into the page behind it. Usually caused by a custom-rolled focus trap. Fix: replace with a native `<dialog>` — native dialog gets focus trap, Esc-close, and `inert`-background for free. The search modal and mobile drawer both follow this pattern; mirror them.

Symptom: `Tab` gets stuck on a closed modal. Usually the `<dialog>` was left in the DOM with `open` still set. Fix: close with `dialog.close()`, not `dialog.removeAttribute('open')`.

### Missing aria-labels on icon-only buttons

axe flags `button-name` or `link-name`. Fix the button, not the rule:

```astro
<button aria-label="Toggle theme" aria-pressed={isDark}>
  <ThemeIcon aria-hidden="true" />
</button>
```

If the glyph is an emoji that screen readers speak ("🌙" → "crescent moon"), keep `aria-label` so the readable name overrides the glyph. Lighthouse's `label-content-name-mismatch` is intentionally downgraded to `warn` for this case — see `.lighthouserc.cjs`.

### Skipped heading levels

Pa11y flags `heading-order`. On MDX content pages, the layout owns the `<h1>` and MDX authors start from `<h2>`. If a chapter body starts with `<h3>` because the author skipped `<h2>`, `scripts/validate-content.mjs` rejects the MDX before CI gets to a11y gates. If a layout itself skips a level, fix the template — do not suppress the rule.

### Table structure

Data tables must have `<caption>`, `<th scope="col">` (or `row`), and an explicit header row. Do not use a table for layout. If Markdown pipe-tables ever render without `<th>` for some reason, wrap them in `<FigureDescribed title="…" description="…">` and add a bordered-row style — but first check the Shiki / remark pipeline output.

## How to add new a11y primitives

New primitives belong in `src/components/a11y/` and go through the full review loop:

1. Read the relevant section of [shared/accessibility.md](../.claude/plans/shared/accessibility.md) — §7 lists the five existing primitives and their contracts.
2. If the primitive wraps an ARIA pattern (tabs, disclosure, combobox), follow the [ARIA Authoring Practices](https://www.w3.org/WAI/ARIA/apg/patterns/) pattern verbatim. Custom ARIA is almost always wrong.
3. Prefer native elements: `<dialog>` beats a modal div, `<details>` beats a custom disclosure, `<button>` beats `role="button"`. Only roll ARIA when a native element does not exist.
4. Ship the primitive with a usage example in the component's frontmatter `@example` block and at least one page that consumes it.
5. Extend `.pa11yci.json` with a URL that exercises the new primitive so it stays under CI after its consumer page lands.

## Escalation — how users report a11y issues

The 404 page and the "Report an issue" CTA in the footer link to a pre-filled GitHub issue template:

- `.github/ISSUE_TEMPLATE/a11y.yml` — fields: page URL, assistive tech + version, expected vs. actual, WCAG criterion (optional).
- Response target: acknowledge within 2 business days; triage to a fix or a won't-fix-because within 5 business days.

If the report describes a barrier on a launch-gate page (home, all-courses, course overview, chapter Full Notes, 404, about, legal), treat it as P0 and block further deploys until patched.

## Things this doc does NOT cover

- **Real-user testing with disabled users.** Spec commitment (quarterly, paid); tracked outside the repo.
- **Dedicated `/a11y` accessibility statement page.** Deferred to Phase 1.5.
- **Dyslexia-friendly font toggle.** The `gyandev:fontFamily` storage slot is reserved; UI is Phase 2.
