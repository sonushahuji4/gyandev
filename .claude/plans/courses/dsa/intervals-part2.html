Sweep Line Algorithm

Meta-Pattern / Template
Sweep Line (Line Sweep)
⚡ Not In Your PDF🔥🔥 CriticalGoogleMetaAmazon
📝 What Is It?

Instead of processing intervals as whole units, decompose each into two events: start (+1) and end (-1). Sort all events, then sweep through, maintaining a running count of active intervals.

🎨 Visual
Intervals: [1,4], [2,5], [7,9] Events: (1,+1) (2,+1) (4,-1) (5,-1) (7,+1) (9,-1) Sweep: Time: 1 2 3 4 5 6 7 8 9 +1 +1 -1 -1 +1 -1 Active: 1 2 2 1 0 0 1 1 0 ↑ peak = 2 (this is your answer for "max overlap")
🖥️ Template
def sweepLine(intervals):
    events = []
    for start, end in intervals:
        events.append((start, 1))
        events.append((end, -1))

    # Sort: by time; if tie, ends (-1) before starts (+1)
    events.sort(key=lambda x: (x[0], x[1]))

    active = max_active = 0
    for _, delta in events:
        active += delta
        max_active = max(max_active, active)

    return max_active
function sweepLine(intervals: number[][]): number {
    const events: [number, number][] = [];
    for (const [start, end] of intervals) {
        events.push([start, 1]);
        events.push([end, -1]);
    }

    events.sort((a, b) => a[0] - b[0] || a[1] - b[1]);

    let active = 0, maxActive = 0;
    for (const [, delta] of events) {
        active += delta;
        maxActive = Math.max(maxActive, active);
    }

    return maxActive;
}

Min Arrows to Burst Balloons

LeetCode #452
Minimum Number of Arrows to Burst Balloons
⚡ Not In Your PDFAmazonMeta
📝 Problem

Balloons are intervals [x_start, x_end]. An arrow at x bursts all balloons where x_start ≤ x ≤ x_end. Find minimum arrows needed.

Input
[[10,16],[2,8],[1,6],[7,12]]
Output
2
💡 Intuition

Sort by end. Shoot at the end of the first balloon. Any balloon that starts after your arrow position needs a new arrow. This is the Activity Selection problem — min arrows = number of non-overlapping groups.

🖥️ Solution
def findMinArrowShots(points):
    if not points: return 0
    points.sort(key=lambda x: x[1])
    arrows, pos = 1, points[0][1]
    for start, end in points[1:]:
        if start > pos:
            arrows += 1
            pos = end
    return arrows
function findMinArrowShots(points: number[][]): number {
    if (points.length === 0) return 0;
    points.sort((a, b) => a[1] - b[1]);
    let arrows = 1, pos = points[0][1];
    for (let i = 1; i < points.length; i++) {
        if (points[i][0] > pos) {
            arrows++;
            pos = points[i][1];
        }
    }
    return arrows;
}

My Calendar III

LeetCode #732
My Calendar III — Max K-Booking
⚡ Not In Your PDFGoogle
📝 Problem

Each call to book(start, end) adds an event. Return the maximum number of concurrent events (the K in "K-booking").

🖥️ Solution — Online Sweep Line
from collections import defaultdict

class MyCalendarThree:
    def __init__(self):
        self.timeline = defaultdict(int)

    def book(self, start, end):
        self.timeline[start] += 1
        self.timeline[end] -= 1
        active = max_k = 0
        for t in sorted(self.timeline):
            active += self.timeline[t]
            max_k = max(max_k, active)
        return max_k
class MyCalendarThree {
    private timeline = new Map<number, number>();

    book(start: number, end: number): number {
        this.timeline.set(start, (this.timeline.get(start) ?? 0) + 1);
        this.timeline.set(end, (this.timeline.get(end) ?? 0) - 1);

        let active = 0, maxK = 0;
        for (const key of [...this.timeline.keys()].sort((a,b) => a-b)) {
            active += this.timeline.get(key)!;
            maxK = Math.max(maxK, active);
        }
        return maxK;
    }
}

Car Pooling

LeetCode #1094
Car Pooling
⚡ Not In Your PDFAmazonGoogleLyft
📝 Problem

Given trips [numPassengers, from, to] and a car with capacity, can all passengers be picked up and dropped off without exceeding capacity?

Input
trips = [[2,1,5],[3,3,7]], capacity = 4
Output
false
🖥️ Solution — Two Approaches
def carPooling(trips, capacity):
    events = []
    for p, s, e in trips:
        events.append((s, p))
        events.append((e, -p))
    events.sort()
    cur = 0
    for _, delta in events:
        cur += delta
        if cur > capacity: return False
    return True
def carPooling(trips, capacity):
    diff = [0] * 1001  # locations 0..1000
    for p, s, e in trips:
        diff[s] += p
        diff[e] -= p
    cur = 0
    for d in diff:
        cur += d
        if cur > capacity: return False
    return True
function carPooling(trips: number[][], capacity: number): boolean {
    const diff = new Array(1001).fill(0);
    for (const [p, s, e] of trips) {
        diff[s] += p;
        diff[e] -= p;
    }
    let cur = 0;
    for (const d of diff) {
        cur += d;
        if (cur > capacity) return false;
    }
    return true;
}

Range Module

LeetCode #715
Range Module — Dynamic Add/Remove/Query
⚡ Not In Your PDFGoogleAmazon
📝 Problem

Implement addRange(left, right), queryRange(left, right), removeRange(left, right) on a dynamic set of intervals.

💡 Intuition

Maintain a sorted list of non-overlapping intervals. For each operation, use binary search to find affected intervals, then merge/split/remove as needed. Combines Insert Interval + Remove Interval patterns.

🖥️ Solution
class RangeModule:
    def __init__(self):
        self.ranges = []

    def addRange(self, left, right):
        new_ranges, placed = [], False
        for s, e in self.ranges:
            if e < left:
                new_ranges.append([s, e])
            elif s > right:
                if not placed:
                    new_ranges.append([left, right])
                    placed = True
                new_ranges.append([s, e])
            else:
                left = min(left, s)
                right = max(right, e)
        if not placed:
            new_ranges.append([left, right])
        self.ranges = new_ranges

    def queryRange(self, left, right):
        for s, e in self.ranges:
            if s <= left and e >= right:
                return True
        return False

    def removeRange(self, left, right):
        new_ranges = []
        for s, e in self.ranges:
            if e <= left or s >= right:
                new_ranges.append([s, e])
            else:
                if s < left: new_ranges.append([s, left])
                if e > right: new_ranges.append([right, e])
        self.ranges = new_ranges
class RangeModule {
    private ranges: number[][] = [];

    addRange(left: number, right: number): void {
        const nr: number[][] = [];
        let placed = false;
        for (const [s, e] of this.ranges) {
            if (e < left) { nr.push([s, e]); }
            else if (s > right) {
                if (!placed) { nr.push([left, right]); placed = true; }
                nr.push([s, e]);
            } else {
                left = Math.min(left, s);
                right = Math.max(right, e);
            }
        }
        if (!placed) nr.push([left, right]);
        this.ranges = nr;
    }

    queryRange(left: number, right: number): boolean {
        return this.ranges.some(([s, e]) => s <= left && e >= right);
    }

    removeRange(left: number, right: number): void {
        const nr: number[][] = [];
        for (const [s, e] of this.ranges) {
            if (e <= left || s >= right) { nr.push([s, e]); }
            else {
                if (s < left) nr.push([s, left]);
                if (e > right) nr.push([right, e]);
            }
        }
        this.ranges = nr;
    }
}

Meeting Rooms III

LeetCode #2402
Meeting Rooms III — Most Used Room
⚡ Not In Your PDF🔥🔥 TrendingGoogleAmazon
📝 Problem

n rooms numbered 0 to n-1. Each meeting gets the lowest-numbered free room. If none free, wait for the earliest to free up. Return the room that held the most meetings.

Input
n = 2, meetings = [[0,10],[1,5],[2,7],[3,4]]
Output
0
🖥️ Solution
import heapq

def mostBooked(n, meetings):
    meetings.sort()
    count = [0] * n
    avail = list(range(n))       # min-heap of available rooms
    busy = []                      # min-heap of (end_time, room)

    for start, end in meetings:
        # Free up finished rooms
        while busy and busy[0][0] <= start:
            _, room = heapq.heappop(busy)
            heapq.heappush(avail, room)

        if avail:
            room = heapq.heappop(avail)
            heapq.heappush(busy, (end, room))
        else:
            free_time, room = heapq.heappop(busy)
            heapq.heappush(busy, (free_time + end - start, room))

        count[room] += 1

    return count.index(max(count))
// Simplified — in production use a proper MinHeap class
function mostBooked(n: number, meetings: number[][]): number {
    meetings.sort((a, b) => a[0] - b[0]);
    const count = new Array(n).fill(0);
    const endTimes = new Array(n).fill(0); // when each room frees up

    for (const [start, end] of meetings) {
        let bestRoom = -1, earliest = Infinity;
        for (let r = 0; r < n; r++) {
            if (endTimes[r] <= start) { bestRoom = r; break; }
            if (endTimes[r] < earliest) { earliest = endTimes[r]; bestRoom = r; }
        }
        const wait = Math.max(0, endTimes[bestRoom] - start);
        endTimes[bestRoom] = start + wait + (end - start);
        count[bestRoom]++;
    }

    return count.indexOf(Math.max(...count));
}

Difference Array

LeetCode #1109
Corporate Flight Bookings / Difference Array
⚡ Not In Your PDFAmazonGoogle
📝 Concept

Instead of updating every element in range [l, r] with value v (O(n) each), do diff[l] += v and diff[r+1] -= v (O(1) each). Prefix sum reconstructs actual values.

Bookings: [[1,2,10], [2,3,20], [2,5,25]] n = 5 diff = [0, 0, 0, 0, 0, 0] After [1,2,10]: [10, 0, -10, 0, 0, 0] After [2,3,20]: [10, 20, -10, -20, 0, 0] After [2,5,25]: [10, 45, -10, -20, 0, -25] Prefix sum: [10, 55, 45, 25, 25]
🖥️ Solution
def corpFlightBookings(bookings, n):
    diff = [0] * (n + 1)
    for first, last, seats in bookings:
        diff[first - 1] += seats
        diff[last] -= seats
    result, running = [], 0
    for i in range(n):
        running += diff[i]
        result.append(running)
    return result
function corpFlightBookings(bookings: number[][], n: number): number[] {
    const diff = new Array(n + 1).fill(0);
    for (const [first, last, seats] of bookings) {
        diff[first - 1] += seats;
        diff[last] -= seats;
    }
    const result: number[] = [];
    let running = 0;
    for (let i = 0; i < n; i++) {
        running += diff[i];
        result.push(running);
    }
    return result;
}

The Skyline Problem

LeetCode #218
The Skyline Problem
⚡ Not In Your PDF🔥🔥 ClassicGoogleMicrosoftMeta
📝 Problem

Given buildings [left, right, height], output the skyline silhouette as a list of key points [x, height].

💡 Intuition — Sweep Line + Max-Heap

Create start/end events for each building. Sweep left to right. Maintain a max-heap of active building heights. When the max height changes → that's a key point in the skyline.

🖥️ Solution
import heapq

def getSkyline(buildings):
    events = []
    for l, r, h in buildings:
        events.append((l, -h, r))   # start: neg height for max-heap
        events.append((r, 0, 0))    # end marker
    events.sort()

    result = [[0, 0]]
    active = [(0, float('inf'))]  # (neg_height, right_edge)

    for x, neg_h, r in events:
        if neg_h:
            heapq.heappush(active, (neg_h, r))
        while active[0][1] <= x:
            heapq.heappop(active)
        max_h = -active[0][0]
        if max_h != result[-1][1]:
            result.append([x, max_h])

    return result[1:]
// Requires a MaxHeap. Conceptual approach shown:
function getSkyline(buildings: number[][]): number[][] {
    const events: [number, number, number][] = [];
    for (const [l, r, h] of buildings) {
        events.push([l, -h, r]);
        events.push([r, 0, 0]);
    }
    events.sort((a, b) => a[0] - b[0] || a[1] - b[1]);

    // Use sorted array to simulate max-heap (simplified)
    const heights: number[] = [0];
    const result: number[][] = [];
    let prevMax = 0;

    for (const [x, negH, r] of events) {
        if (negH < 0) heights.push(-negH);
        else { const idx = heights.indexOf(r); if(idx>-1) heights.splice(idx,1); }
        const curMax = Math.max(...heights);
        if (curMax !== prevMax) {
            result.push([x, curMax]);
            prevMax = curMax;
        }
    }
    return result;
}

Rectangle Area II, Interval Tree, Task Scheduler

LeetCode #850
P20: Rectangle Area II — Coordinate Compression + Sweep
⚡ Not In Your PDFGoogle
💡 Core Idea

Compress y-coordinates, create open/close events sorted by x. Sweep left-to-right: at each x, compute the total active y-coverage, multiply by the x-delta. This extends 1D sweep line into 2D.

def rectangleArea(rectangles):
    MOD = 10**9 + 7
    events = []
    for x1, y1, x2, y2 in rectangles:
        events.append((x1, 0, y1, y2))
        events.append((x2, 1, y1, y2))
    events.sort()
    active, area, prev_x = [], 0, events[0][0]
    for x, typ, y1, y2 in events:
        y_cover = cur = 0
        for ay1, ay2 in sorted(active):
            cur = max(cur, ay1)
            y_cover += max(0, ay2 - cur)
            cur = max(cur, ay2)
        area += y_cover * (x - prev_x)
        prev_x = x
        if typ == 0: active.append((y1, y2))
        else: active.remove((y1, y2))
    return area % MOD
// Same algorithm in TypeScript — sweep x-events, track active y-intervals
function rectangleArea(rects: number[][]): number {
    const MOD = 1e9 + 7;
    const events: [number,number,number,number][] = [];
    for (const [x1,y1,x2,y2] of rects) {
        events.push([x1,0,y1,y2]);
        events.push([x2,1,y1,y2]);
    }
    events.sort((a,b) => a[0]-b[0] || a[1]-b[1]);
    const active: [number,number][] = [];
    let area = 0, prevX = events[0][0];
    for (const [x,typ,y1,y2] of events) {
        let yCov = 0, cur = 0;
        for (const [a,b] of [...active].sort((a,b)=>a[0]-b[0])) {
            cur = Math.max(cur, a);
            yCov += Math.max(0, b - cur);
            cur = Math.max(cur, b);
        }
        area = (area + yCov * (x - prevX)) % MOD;
        prevX = x;
        if (typ===0) active.push([y1,y2]);
        else { const i=active.findIndex(([a,b])=>a===y1&&b===y2); active.splice(i,1); }
    }
    return area;
}
Advanced Data Structure
P21: Interval Tree
⚡ Not In Your PDFGoogleMicrosoft
💡 When Do You Need This?

When you need to efficiently answer: "Which intervals overlap with a given query point or range?" This gives O(log n + k) per query vs O(n) brute force. Rarely coded from scratch in interviews, but knowing the concept shows depth.

How it works: Choose a center point (median of all endpoints). Intervals crossing the center are stored at this node (sorted by both start and end). Intervals entirely left go to the left subtree; entirely right go to the right subtree. To query a point: check center intervals, then recurse into the appropriate child.

LeetCode #621
P22: Task Scheduler
⚡ Not In Your PDFMetaGoogleAmazon
📝 Problem

Given tasks and cooldown n, find minimum time to finish all. Same tasks must be ≥ n apart.

Input
tasks = ["A","A","A","B","B","B"], n = 2
Output
8
from collections import Counter

def leastInterval(tasks, n):
    freq = Counter(tasks)
    max_f = max(freq.values())
    max_count = sum(1 for v in freq.values() if v == max_f)
    return max(len(tasks), (max_f - 1) * (n + 1) + max_count)
function leastInterval(tasks: string[], n: number): number {
    const freq = new Map<string, number>();
    for (const t of tasks) freq.set(t, (freq.get(t) ?? 0) + 1);
    const maxF = Math.max(...freq.values());
    const maxCount = [...freq.values()].filter(v => v === maxF).length;
    return Math.max(tasks.length, (maxF - 1) * (n + 1) + maxCount);
}

Ultimate Cheatsheet

🧠 6 Core Techniques

① Sort + Merge

Sort by start, walk + merge. The bread and butter. ~50% of problems.

② Greedy (Sort by End)

Finish earliest first. For optimization: max kept, min removed, min arrows.

③ Sweep Line

Break into +1/-1 events, sweep. For counting overlaps, peaks, skyline.

④ Min-Heap

Track end times for resource allocation. "Is any room free?" in O(log n).

⑤ Two Pointers

Two sorted lists → advance pointer with smaller end. For intersection.

⑥ Difference Array

O(1) range updates. Prefix sum to reconstruct. Best for bounded integers.

⚡ 6 Pro Tips for FAANG
1. Always sort first — state this immediately. 95% of interval problems need it.
2. Clarify interval type — "Closed [a,b] or half-open [a,b)?" Shows attention to detail.
3. Know multiple approaches — For Meeting Rooms II, know heap + sweep + two-pointer. Handle follow-ups.
4. Sort by START for merging, END for greedy — This is the #1 mistake. Never confuse them.
5. "Maximum concurrent" = sweep line — Peak usage, max overlap, resource capacity → sweep.
6. Bounded integers → difference array — Coordinates 0-1000? Skip sweep line, use diff array for O(1) updates.
