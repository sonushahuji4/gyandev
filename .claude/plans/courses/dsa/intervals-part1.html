Chapter 01

Foundational Concepts

Every interval problem starts here. Master these concepts before touching a single LeetCode problem.

Foundation
What is an Interval?

An interval [start, end] represents a continuous range. In interviews, intervals model time ranges (meetings, schedules), spatial ranges (coordinates, balloons), or abstract number-line ranges.

📐 Types of Intervals
Closed [a, b] → includes both a and b Open (a, b) → excludes both a and b Half-Open [a, b) → includes a, excludes b ← MOST COMMON in real systems Half-Open (a, b] → excludes a, includes b
FAANG Reality Check: LeetCode uses closed intervals [a, b]. But real-world systems (Google Calendar, cron jobs, database ranges) use half-open intervals [a, b) because they compose cleanly — [9, 10) and [10, 11) are adjacent, not overlapping. Always clarify with your interviewer which type they mean.
🔑 The Universal Overlap Formula

Two intervals [a1, a2] and [b1, b2] overlap if and only if:

max(a1, b1) ≤ min(a2, b2) If true: Intersection = [max(a1, b1), min(a2, b2)] Union = [min(a1, b1), max(a2, b2)] If false: No overlap — the gap between them is [min(a2,b2), max(a1,b1)]
Why this formula matters: Instead of checking 6 separate cases, this single expression handles ALL overlap scenarios. Memorize max(start1, start2) <= min(end1, end2) — it's the foundation of 80% of interval solutions.
Chapter 02

The 6 Interval Relationships

Every possible arrangement of two intervals falls into one of these six categories. Understanding them intuitively is key to writing clean, bug-free code.

① A Entirely Before B

A: ■■■■■ B: ■■■■■ ───┬───┬─────┬───┬───→ A.s A.e B.s B.e
Condition: A.end < B.start
Result: No overlap — both intervals stay separate.
Think of it as: A finishes before B even begins.

② B Entirely Before A

A: ■■■■■ B: ■■■■■ ───┬───┬─────┬───┬───→ B.s B.e A.s A.e
Condition: B.end < A.start
Result: No overlap — mirror of case ①.
After sorting by start, this case becomes impossible — A always starts ≤ B.

③ Partial Overlap (A starts first)

A: ■■■■■■■■■ B: ■■■■■■■■■ ▓▓▓▓▓ ← intersection ──┬───────┬─────┬────→ A.s B.s A.e B.e
Condition: A.start ≤ B.start AND A.end ≥ B.start AND A.end ≤ B.end
Merged: [A.start, B.end]
Intersection: [B.start, A.end]

④ Partial Overlap (B starts first)

A: ■■■■■■■■■ B: ■■■■■■■■■ ▓▓▓▓▓ ← intersection ──┬───────┬─────┬────→ B.s A.s B.e A.e
Condition: B.start ≤ A.start AND B.end ≥ A.start AND B.end ≤ A.end
Merged: [B.start, A.end]
After sorting by start, this case also becomes impossible.

⑤ A Fully Contains B

A: ■■■■■■■■■■■■■■■ B: ■■■■■■■ ──┬──────────────┬───→ A.s B.s B.e A.e
Condition: A.start ≤ B.start AND A.end ≥ B.end
Merged: [A.start, A.end] (A unchanged — it already covers B)
Trap: Don't forget to update end as max(A.end, B.end) in code — it handles this naturally.

⑥ B Fully Contains A

A: ■■■■■■■ B: ■■■■■■■■■■■■■■■ ──┬──────────────┬───→ B.s A.s A.e B.e
Condition: B.start ≤ A.start AND B.end ≥ A.end
Merged: [B.start, B.end]
After sorting, this becomes: A.start = B.start but B.end > A.end.
🧠 The Key Insight That Simplifies Everything
After sorting by start time, you reduce 6 cases → 3 cases:

Since A always starts ≤ B after sorting, cases ② and ④ (B starts first) become impossible. Your code only needs to handle:

1. No overlap: A.end < B.start → keep both separate
2. Partial overlap: A.end ≥ B.start AND A.end < B.end → merge to [A.start, B.end]
3. Containment: A.end ≥ B.end → keep A as-is (it already covers B)

Cases 2 and 3 collapse into a single line: merged_end = max(A.end, B.end)
💡 My Mental Model

Think of intervals as blocks on a timeline ruler. Sorting aligns them left-to-right. Then you just walk right, and for each new block, ask: "does it touch or overlap the last block I placed?" If yes → extend. If no → start a new block. That's it. That single mental model solves Merge Intervals, Employee Free Time, and several others.

Chapter 03

Sorting Strategies

The most important decision in any interval problem: how to sort. The wrong sort order = wrong answer.

Sort by Start Time

sort(key=λ x: x[0])

Use for: Merging, inserting, intersection, conflict detection

Problems: #56, #57, #252, #253, #759, #986

Sort by End Time

sort(key=λ x: x[1])

Use for: Greedy selection — max non-overlapping, min removals, min arrows

Problems: #435, #452, Activity Selection

Sort Start ↑, then End ↓

sort(key=λ x: (x[0], -x[1]))

Use for: Remove Covered Intervals — process largest-span first at each start

Problems: #1288

Event-Based (Sweep Line)

events.sort(key=λ x: (x[0], x[1]))

Use for: Counting active intervals at each point, peak usage, skyline

Problems: #253 alt, #732, #1094, #218

The #1 Mistake: Using sort-by-start when the problem needs sort-by-end (or vice versa). Rule of thumb — if the problem says "maximize" or "minimize" a count, you almost certainly need sort by end (greedy). If it says "merge" or "combine", you need sort by start.
Chapter 04

Concepts, Tips & Common Approaches

Patterns and mental shortcuts that help you solve any interval problem, even ones you've never seen before.

🧩 The 5 Common Approaches

1. Sort + Linear Scan

Sort intervals, then walk through them maintaining the "last" interval. Compare each new interval with the last — merge, extend, or start fresh. Solves ~50% of all interval problems.

2. Sweep Line (Events)

Decompose intervals into start/end events. Sort events. Sweep through maintaining a counter (or heap). Answers questions about "how many overlap at any point?"

3. Greedy Selection

Sort by end time, greedily pick intervals that finish earliest. For optimization: maximize kept intervals, minimize removals, minimize resources.

4. Priority Queue (Heap)

Track active intervals by end time using a min-heap. Enables efficient "is any room free?" checks. Core technique for resource allocation problems.

5. Two Pointers

When processing two already-sorted interval lists simultaneously. Advance the pointer whose interval ends first. Key for intersection problems.

🎯 Pattern Recognition Shortcuts
"Merge overlapping" → Sort by start + linear scan
"Can attend all?" / "Any conflict?" → Sort + check consecutive pairs
"How many rooms/resources?" → Sweep line or min-heap
"Max events to attend" → Sort by end (greedy)
"Min removals" → Sort by end (greedy) — answer = n - max_kept
"Intersection of two lists" → Two pointers
"Free time / gaps" → Merge all → find gaps between merged
"Range update on array" → Difference array
🛡️ Universal Edge Cases Checklist
Always ask yourself these before coding:

1. Empty input? → Return empty / 0 / true depending on problem
2. Single interval? → Usually the trivial base case
3. All intervals identical? → They all merge into one
4. No overlaps at all? → Output equals input
5. One giant interval covering everything? → Everything merges into it
6. Touching endpoints [1,2] and [2,3]? → Overlapping or not? Clarify!
7. Start = End? [5,5]? → Zero-length interval — valid or degenerate?
8. Negative coordinates? → Does your solution handle them?
9. Very large values? → Integer overflow in start + end calculations?
Pattern 01

Merge Intervals

LeetCode #56
Merge Intervals
🔥 Very High Frequency GoogleMetaAmazonMicrosoft
📝 Problem Statement

Given an array of intervals where intervals[i] = [startᵢ, endᵢ], merge all overlapping intervals, and return an array of the non-overlapping intervals that cover all the ranges in the input.

Input
intervals = [[1,3],[2,6],[8,10],[15,18]]
Output
[[1,6],[8,10],[15,18]]
Input
intervals = [[1,4],[4,5]]
Output
[[1,5]]
💡 Intuition

Sort by start time so overlapping intervals become neighbors. Walk left to right: if the current interval's start ≤ the last merged interval's end, they overlap — extend the end. Otherwise, it's a new group.

🎨 Visual Walkthrough
Input: [[1,3], [2,6], [8,10], [15,18]] After sort by start: [[1,3], [2,6], [8,10], [15,18]] (already sorted) Step 1: merged = [[1,3]] Step 2: [2,6] → 2 ≤ 3? YES → merge → end = max(3,6) = 6 → [[1,6]] Step 3: [8,10] → 8 ≤ 6? NO → new group → [[1,6], [8,10]] Step 4: [15,18] → 15 ≤ 10? NO → new group → [[1,6], [8,10], [15,18]] Timeline: 1──██████──6 8─███─10 15─████─18
🖥️ Solution
def merge(intervals: list[list[int]]) -> list[list[int]]:
    if not intervals:
        return []

    intervals.sort(key=lambda x: x[0])
    merged = [intervals[0]]

    for start, end in intervals[1:]:
        if start <= merged[-1][1]:
            merged[-1][1] = max(merged[-1][1], end)
        else:
            merged.append([start, end])

    return merged
function merge(intervals: number[][]): number[][] {
    if (intervals.length === 0) return [];

    intervals.sort((a, b) => a[0] - b[0]);
    const merged: number[][] = [intervals[0]];

    for (let i = 1; i < intervals.length; i++) {
        const last = merged[merged.length - 1];
        const [start, end] = intervals[i];

        if (start <= last[1]) {
            last[1] = Math.max(last[1], end);
        } else {
            merged.push([start, end]);
        }
    }

    return merged;
}

O(n log n)

🛡️ Edge Case Thinking
Empty array: Return []. Always guard this first.
Single interval: Return it as-is — nothing to merge with.
All overlapping: e.g. [[1,10],[2,5],[3,7]] → everything merges into [[1,10]]. Your max() in the merge step handles containment automatically.
Touching endpoints: [1,2] and [2,3] → overlap because 2 ≤ 2. The ≤ (not <) is critical.
Already sorted & non-overlapping: Output = input. The else branch handles this.
Pattern 02

Insert Interval

LeetCode #57
Insert Interval
🔥 High Frequency GoogleMetaLinkedIn
📝 Problem Statement

Given a sorted list of non-overlapping intervals, insert a new interval. Merge if necessary. Return the result still sorted and non-overlapping.

Input
intervals = [[1,2],[3,5],[6,7],[8,10],[12,16]]
newInterval = [4,8]
Output
[[1,2],[3,10],[12,16]]
💡 Intuition

Three-phase sweep: (1) Collect all intervals that end before the new one starts — no overlap, keep as-is. (2) Merge all intervals that overlap with the new one — expand the new interval. (3) Append remaining intervals after the merged zone. No sorting needed since input is pre-sorted!

🎨 Visual Walkthrough
intervals: [1,2] [3,5] [6,7] [8,10] [12,16] new: [====4========8====] Phase 1 — Before: [1,2] ends at 2 < 4 → keep [1,2] Phase 2 — Merge: [3,5] starts 3 ≤ 8 → merge → new = [3, 8] [6,7] starts 6 ≤ 8 → merge → new = [3, 8] [8,10] starts 8 ≤ 8 → merge → new = [3, 10] → add [3, 10] Phase 3 — After: [12,16] → keep [12,16] Result: [[1,2], [3,10], [12,16]]
🖥️ Solution
def insert(intervals: list[list[int]], newInterval: list[int]) -> list[list[int]]:
    result = []
    i, n = 0, len(intervals)

    # Phase 1: intervals entirely before newInterval
    while i < n and intervals[i][1] < newInterval[0]:
        result.append(intervals[i])
        i += 1

    # Phase 2: merge overlapping intervals
    while i < n and intervals[i][0] <= newInterval[1]:
        newInterval[0] = min(newInterval[0], intervals[i][0])
        newInterval[1] = max(newInterval[1], intervals[i][1])
        i += 1
    result.append(newInterval)

    # Phase 3: remaining intervals
    while i < n:
        result.append(intervals[i])
        i += 1

    return result
function insert(intervals: number[][], newInterval: number[]): number[][] {
    const result: number[][] = [];
    let i = 0;
    const n = intervals.length;

    // Phase 1: before
    while (i < n && intervals[i][1] < newInterval[0]) {
        result.push(intervals[i]);
        i++;
    }

    // Phase 2: merge
    while (i < n && intervals[i][0] <= newInterval[1]) {
        newInterval[0] = Math.min(newInterval[0], intervals[i][0]);
        newInterval[1] = Math.max(newInterval[1], intervals[i][1]);
        i++;
    }
    result.push(newInterval);

    // Phase 3: after
    while (i < n) {
        result.push(intervals[i]);
        i++;
    }

    return result;
}

New interval before everything: [0,0] into [[1,5]] → Phase 1 does nothing, Phase 2 does nothing, result = [[0,0],[1,5]].

New interval swallows everything: Phase 2 merges all existing intervals into one giant merged interval.

Pattern 03

Meeting Rooms I — Conflict Detection

LeetCode #252
Can Attend All Meetings?
AmazonMetaBloomberg
📝 Problem Statement

Given an array of meeting time intervals, determine if a person can attend all meetings (i.e., no two meetings overlap).

Input
[[0,30],[5,10],[15,20]]
Output
false
Input
[[7,10],[2,4]]
Output
true
💡 Intuition

Sort by start time. After sorting, overlapping meetings must be adjacent. So just check each consecutive pair: if any meeting starts before the previous one ends → conflict.

🖥️ Solution
def canAttendMeetings(intervals: list[list[int]]) -> bool:
    intervals.sort(key=lambda x: x[0])

    for i in range(1, len(intervals)):
        if intervals[i][0] < intervals[i - 1][1]:
            return False

    return True
function canAttendMeetings(intervals: number[][]): boolean {
    intervals.sort((a, b) => a[0] - b[0]);

    for (let i = 1; i < intervals.length; i++) {
        if (intervals[i][0] < intervals[i - 1][1]) {
            return false;
        }
    }

    return true;
}
Metric
Value
Reason
Time
O(n log n)
Dominated by sorting; linear scan is O(n)
Space
O(1)
In-place sort, no extra structure
🛡️ Edge Case Thinking
0 or 1 meetings: Always true — no possible conflict.
[1,2] and [2,3]: Does 2 < 2 fail? No → no conflict. These are back-to-back meetings (half-open semantics). If closed semantics, you'd use ≤ instead.
All same time: [[1,5],[1,5],[1,5]] → first pair already conflicts → return false.

Meeting Rooms II — Minimum Rooms

LeetCode #253
Minimum Conference Rooms Required
🔥🔥 Very High Frequency GoogleAmazonMetaMicrosoftApple
📝 Problem Statement

Given meeting time intervals, find the minimum number of conference rooms required to hold all meetings.

Input
[[0,30],[5,10],[15,20]]
Output
2
Input
[[7,10],[2,4]]
Output
1
💡 Intuition — Three Solutions (Know ALL for FAANG)
FAANG interviewers love follow-ups. They'll ask you to solve it one way, then say "Can you do it differently?" Knowing three approaches signals deep mastery.
Solution A: Min-Heap (Most Intuitive)
import heapq

def minMeetingRooms(intervals: list[list[int]]) -> int:
    if not intervals:
        return 0

    intervals.sort(key=lambda x: x[0])

    # Heap stores end times of active rooms
    heap = [intervals[0][1]]

    for i in range(1, len(intervals)):
        # If earliest-ending room is free, reuse it
        if intervals[i][0] >= heap[0]:
            heapq.heappop(heap)
        # Assign a room (new or reused)
        heapq.heappush(heap, intervals[i][1])

    return len(heap)
// TypeScript needs a MinHeap. Using sorted array for clarity:
function minMeetingRooms(intervals: number[][]): number {
    if (intervals.length === 0) return 0;

    intervals.sort((a, b) => a[0] - b[0]);
    const endTimes: number[] = [intervals[0][1]];

    for (let i = 1; i < intervals.length; i++) {
        endTimes.sort((a, b) => a - b); // simulate min-heap
        if (intervals[i][0] >= endTimes[0]) {
            endTimes.shift();
        }
        endTimes.push(intervals[i][1]);
    }

    return endTimes.length;
}
Solution B: Sweep Line (Most Elegant)
def minMeetingRooms(intervals: list[list[int]]) -> int:
    events = []
    for start, end in intervals:
        events.append((start, 1))    # meeting starts
        events.append((end, -1))     # meeting ends

    events.sort(key=lambda x: (x[0], x[1]))

    max_rooms = current = 0
    for _, delta in events:
        current += delta
        max_rooms = max(max_rooms, current)

    return max_rooms
function minMeetingRooms(intervals: number[][]): number {
    const events: [number, number][] = [];
    for (const [start, end] of intervals) {
        events.push([start, 1]);
        events.push([end, -1]);
    }

    events.sort((a, b) => a[0] - b[0] || a[1] - b[1]);

    let maxRooms = 0, current = 0;
    for (const [, delta] of events) {
        current += delta;
        maxRooms = Math.max(maxRooms, current);
    }

    return maxRooms;
}

Pattern 05

Non-Overlapping Intervals

LeetCode #435
Minimum Removals to Make Non-Overlapping
AmazonGoogleMicrosoft
📝 Problem Statement

Given an array of intervals, return the minimum number of intervals you need to remove to make the rest non-overlapping.

Input
[[1,2],[2,3],[3,4],[1,3]]
Output
1
Input
[[1,2],[1,2],[1,2]]
Output
2
💡 Intuition — Sort by END (Greedy)
This is the Activity Selection Problem in disguise. By sorting by end time, we greedily keep intervals that finish earliest, leaving maximum room for future intervals. The number of removals = total - max intervals we can keep.
🎨 Visual Walkthrough
Input: [[1,2], [2,3], [3,4], [1,3]] After sort by END: [[1,2], [2,3], [1,3], [3,4]] Step 1: keep [1,2], last_end = 2 Step 2: [2,3] → 2 ≥ 2 → no overlap → KEEP, last_end = 3 Step 3: [1,3] → 1 < 3 → overlap → REMOVE, count = 1 Step 4: [3,4] → 3 ≥ 3 → no overlap → KEEP Answer: 1 removal
🖥️ Solution
def eraseOverlapIntervals(intervals: list[list[int]]) -> int:
    if not intervals:
        return 0

    # KEY: Sort by END time for greedy
    intervals.sort(key=lambda x: x[1])

    count = 0
    last_end = intervals[0][1]

    for i in range(1, len(intervals)):
        if intervals[i][0] < last_end:
            count += 1     # overlap → remove this one
        else:
            last_end = intervals[i][1]

    return count
function eraseOverlapIntervals(intervals: number[][]): number {
    if (intervals.length === 0) return 0;

    intervals.sort((a, b) => a[1] - b[1]);

    let count = 0;
    let lastEnd = intervals[0][1];

    for (let i = 1; i < intervals.length; i++) {
        if (intervals[i][0] < lastEnd) {
            count++;
        } else {
            lastEnd = intervals[i][1];
        }
    }

    return count;
}

Why sort by end, not start? If you sort by start and see [1,100] first, you'd keep it — blocking everything. Sort by end ensures you keep the interval that "gets out of the way" fastest.
All identical intervals: [[1,2],[1,2],[1,2]] → keep 1, remove 2.
No overlaps: count stays 0 → return 0.
Pattern 06

Interval List Intersections

LeetCode #986
Interval List Intersections
🔥 High FrequencyGoogleMetaUber
📝 Problem Statement

Given two lists of closed, sorted, disjoint intervals, return their intersection.

Input
firstList = [[0,2],[5,10],[13,23],[24,25]]
secondList = [[1,5],[8,12],[15,24],[25,26]]
Output
[[1,2],[5,5],[8,10],[15,23],[24,24],[25,25]]
💡 Intuition

Two pointers, one per list. At each step: compute intersection of the current pair using [max(starts), min(ends)]. If valid → add it. Then advance the pointer whose interval ends first (it can't intersect anything else from the other list).

🖥️ Solution
def intervalIntersection(A: list, B: list) -> list:
    result = []
    i = j = 0

    while i < len(A) and j < len(B):
        lo = max(A[i][0], B[j][0])
        hi = min(A[i][1], B[j][1])

        if lo <= hi:
            result.append([lo, hi])

        # Advance the pointer with the smaller end
        if A[i][1] < B[j][1]:
            i += 1
        else:
            j += 1

    return result
function intervalIntersection(A: number[][], B: number[][]): number[][] {
    const result: number[][] = [];
    let i = 0, j = 0;

    while (i < A.length && j < B.length) {
        const lo = Math.max(A[i][0], B[j][0]);
        const hi = Math.min(A[i][1], B[j][1]);

        if (lo <= hi) result.push([lo, hi]);

        if (A[i][1] < B[j][1]) i++;
        else j++;
    }

    return result;
}

Reason
Time
O(m + n)
Each pointer advances at most m or n times
Space
O(m + n)

🛡️ Edge Case Thinking
Either list empty: Return [] — no intersection possible.
Point intersection [5,5]: Valid! lo = hi = 5, and 5 ≤ 5 is true.
No overlap at all: lo > hi every time, result stays empty.
Pattern 07

Employee Free Time

LeetCode #759
Employee Free Time
GoogleUberAirbnb
📝 Problem Statement

Given the work schedules of all employees (list of lists of intervals), find the common free time for all employees — gaps when nobody is working.

Input
schedule = [[[1,2],[5,6]], [[1,3]], [[4,10]]]
Output
[[3,4]]
💡 Intuition

Flatten all schedules into one list → sort → merge → gaps between merged intervals = free time. It's literally "Merge Intervals" + "find the gaps."

🖥️ Solution
def employeeFreeTime(schedule):
    # Flatten all intervals
    all_intervals = []
    for employee in schedule:
        all_intervals.extend(employee)

    all_intervals.sort(key=lambda x: x[0])

    # Merge
    merged = [all_intervals[0]]
    for interval in all_intervals[1:]:
        if interval[0] <= merged[-1][1]:
            merged[-1][1] = max(merged[-1][1], interval[1])
        else:
            merged.append(interval)

    # Gaps = free time
    free = []
    for i in range(1, len(merged)):
        free.append([merged[i-1][1], merged[i][0]])

    return free
function employeeFreeTime(schedule: number[][][]): number[][] {
    const all: number[][] = schedule.flat();
    all.sort((a, b) => a[0] - b[0]);

    const merged: number[][] = [all[0]];
    for (let i = 1; i < all.length; i++) {
        const last = merged[merged.length - 1];
        if (all[i][0] <= last[1]) {
            last[1] = Math.max(last[1], all[i][1]);
        } else {
            merged.push([...all[i]]);
        }
    }

    const free: number[][] = [];
    for (let i = 1; i < merged.length; i++) {
        free.push([merged[i - 1][1], merged[i][0]]);
    }

    return free;
}

Value
Reason
Time
O(N log N)
N = total intervals across all employees
Space

🛡️ Edge Case Thinking
Single employee: Their gaps are the free time. But if we're looking for COMMON free time across ALL employees, a single employee's schedule has no "common" concept — depends on problem semantics.
Everyone busy at all times: merged = one continuous block → no gaps → return [].

Pattern 08

Remove Interval

LeetCode #1272
Remove Interval
Google
📝 Problem Statement

Given a sorted set of disjoint intervals and an interval to remove, return the remaining intervals after removal.

Input
intervals = [[0,2],[3,4],[5,7]], toBeRemoved = [1,6]
Output
[[0,1],[6,7]]
💡 Intuition

For each interval, check: does it overlap with the removal range? If no → keep. If yes → keep the parts outside the removal zone (left remnant and/or right remnant).

🎨 Visual
Interval: [══════════════════════] Remove: [════════════] Left part: [══════] Right part: [══════]
🖥️ Solution
def removeInterval(intervals, toBeRemoved):
    result = []
    rm_s, rm_e = toBeRemoved

    for s, e in intervals:
        if e <= rm_s or s >= rm_e:
            result.append([s, e])       # no overlap
        else:
            if s < rm_s:
                result.append([s, rm_s])  # left remnant
            if e > rm_e:
                result.append([rm_e, e])  # right remnant

    return result
function removeInterval(intervals: number[][], toBeRemoved: number[]): number[][] {
    const result: number[][] = [];
    const [rmS, rmE] = toBeRemoved;

    for (const [s, e] of intervals) {
        if (e <= rmS || s >= rmE) {
            result.push([s, e]);
        } else {
            if (s < rmS) result.push([s, rmS]);
            if (e > rmE) result.push([rmE, e]);
        }
    }

    return result;
}

My Calendar I

LeetCode #729
My Calendar I — No Double Booking
Google
📝 Problem Statement

Implement a calendar where you can add events. A new event can be added if it doesn't cause a double booking (no overlap with existing events). Events are half-open intervals [start, end).

Example
book(10,20) → true, book(15,25) → false, book(20,30) → true
💡 Intuition

For each new booking, check against all existing ones using the overlap formula: start < existing_end AND end > existing_start. If any overlap → reject. Otherwise → add.

🖥️ Solution
class MyCalendar:
    def __init__(self):
        self.bookings = []

    def book(self, start: int, end: int) -> bool:
        for s, e in self.bookings:
            if start < e and end > s:  # overlap check
                return False
        self.bookings.append((start, end))
        return True

# book() → O(n) per call.  Optimized: SortedList → O(log n)
class MyCalendar {
    private bookings: [number, number][] = [];

    book(start: number, end: number): boolean {
        for (const [s, e] of this.bookings) {
            if (start < e && end > s) return false;
        }
        this.bookings.push([start, end]);
        return true;
    }
}

My Calendar II

LeetCode #731
My Calendar II — Allow Double, Block Triple
Google
📝 Problem Statement

Implement a calendar that allows double booking but rejects triple booking.

💡 Intuition

Maintain two lists: bookings (all events) and overlaps (double-booked regions). For each new event: if it overlaps any existing double-booking → would cause triple → reject. Otherwise, record any new double-booking regions it creates, then add it.

🖥️ Solution
class MyCalendarTwo:
    def __init__(self):
        self.bookings = []
        self.overlaps = []  # double-booked regions

    def book(self, start: int, end: int) -> bool:
        # Check against triple booking
        for os, oe in self.overlaps:
            if start < oe and end > os:
                return False

        # Record new double-bookings
        for bs, be in self.bookings:
            if start < be and end > bs:
                self.overlaps.append((max(start, bs), min(end, be)))

        self.bookings.append((start, end))
        return True
class MyCalendarTwo {
    private bookings: [number, number][] = [];
    private overlaps: [number, number][] = [];

    book(start: number, end: number): boolean {
        for (const [os, oe] of this.overlaps) {
            if (start < oe && end > os) return false;
        }

        for (const [bs, be] of this.bookings) {
            if (start < be && end > bs) {
                this.overlaps.push([Math.max(start, bs), Math.min(end, be)]);
            }
        }

        this.bookings.push([start, end]);
        return true;
    }
}

Data Stream as Disjoint Intervals

LeetCode #352
Data Stream as Disjoint Intervals
Google
📝 Problem Statement

Given a stream of integers, maintain a summary of the numbers seen as a list of disjoint intervals. Implement addNum(value) and getIntervals().

Example
addNum(1) → [[1,1]]
addNum(3) → [[1,1],[3,3]]
addNum(7) → [[1,1],[3,3],[7,7]]
addNum(2) → [[1,3],[7,7]] (1,2,3 merge!)
addNum(6) → [[1,3],[6,7]]
💡 Intuition

Use binary search to find where the new number fits. Check if it can merge with left neighbor (value - 1 ≤ left.end), right neighbor (value + 1 ≥ right.start), or both.

🖥️ Solution
import bisect

class SummaryRanges:
    def __init__(self):
        self.intervals = []

    def addNum(self, value: int) -> None:
        intervals = self.intervals
        idx = bisect.bisect_left(intervals, [value, value])

        merge_l = idx > 0 and intervals[idx-1][1] >= value - 1
        merge_r = idx < len(intervals) and intervals[idx][0] <= value + 1

        if merge_l and merge_r:
            intervals[idx-1][1] = max(intervals[idx-1][1], intervals[idx][1])
            intervals.pop(idx)
        elif merge_l:
            intervals[idx-1][1] = max(intervals[idx-1][1], value)
        elif merge_r:
            intervals[idx][0] = min(intervals[idx][0], value)
        else:
            intervals.insert(idx, [value, value])

    def getIntervals(self):
        return self.intervals
class SummaryRanges {
    private intervals: number[][] = [];

    addNum(value: number): void {
        const iv = this.intervals;
        // Binary search for insertion point
        let lo = 0, hi = iv.length;
        while (lo < hi) {
            const mid = (lo + hi) >> 1;
            if (iv[mid][0] < value) lo = mid + 1;
            else hi = mid;
        }
        const idx = lo;

        const mergeL = idx > 0 && iv[idx - 1][1] >= value - 1;
        const mergeR = idx < iv.length && iv[idx][0] <= value + 1;

        if (mergeL && mergeR) {
            iv[idx - 1][1] = Math.max(iv[idx - 1][1], iv[idx][1]);
            iv.splice(idx, 1);
        } else if (mergeL) {
            iv[idx - 1][1] = Math.max(iv[idx - 1][1], value);
        } else if (mergeR) {
            iv[idx][0] = Math.min(iv[idx][0], value);
        } else {
            iv.splice(idx, 0, [value, value]);
        }
    }

    getIntervals(): number[][] {
        return this.intervals;
    }
}

🛡️ Edge Case Thinking
Duplicate values: addNum(5) twice — second call finds 5 already within an existing interval, so nothing changes.
Sequential values: addNum(1), addNum(2), addNum(3) — each merges with previous → final = [[1,3]].
Bridging gap: [[1,1],[3,3]] + addNum(2) → merges both neighbors into [[1,3]].
