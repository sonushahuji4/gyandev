# Implementation Plan: 404 Page

## 1. Overview

Smallest page in Phase 1: a static HTML "not found" fallback that Cloudflare Pages serves with HTTP 404 for any unmatched URL. The page owns nothing new conceptually — it composes existing shell primitives (`PageShell`, `SEO`, `SkipLink`, `LiveRegion`), triggers the pre-built `SearchModal` for lookups, and links to three anchor destinations (Home, All Courses, the featured course). We introduce one new component (`ErrorHero`) scoped to this page. A post-build check asserts Cloudflare Pages serves `dist/404.html` with status 404. Goal: a friendly recovery point that ships under 30 KB total, loads before a user finishes reading the headline, and blocks search engines from indexing the 404 response itself.

## 2. Spec Reference

See `.claude/specs/pages/404.md`. Load-bearing requirements:

- §1 Four jobs: acknowledge, redirect, help-find (search), preserve trust.
- §3 Sections: Hero with "4 0 4" display + friendly headline → search box → quick links (Home, All Courses, featured) → "Broken link?" GitHub link.
- §4 Layout: ErrorLayout — no sidebars, no TOC, centered single column.
- §7 HTTP status 404 (not 200) — Cloudflare Pages serves `/404.html` as the fallback.
- §8 Meta: `<title>Page Not Found — GyanDev</title>`, `<meta name="robots" content="noindex, follow">`, canonical `/404`.
- §9 No structured data (per spec — intentional).
- §10 Page weight < 30 KB, zero required JS.
- §11 H1 = "This page wandered off" (NOT "404" — SR pronounces it oddly). Search input labeled "Search GyanDev".
- §12 Success: HTTP 404, shared chrome present, search works, links work, mobile friendly, intentional `noindex` signal.

## 3. Technical Approach

**3.1 Single route, flat HTML.** `src/pages/404.astro` renders a complete standalone page. With `trailingSlash: 'never'` + `build.format: 'file'` from `.claude/plans/shared/routing-and-urls.md`, Astro emits `dist/404.html` — Cloudflare Pages automatically serves that file with HTTP 404 status for any unmatched URL (documented behavior: https://developers.cloudflare.com/pages/configuration/serving-pages/#not-found-behavior). No `_redirects` entry needed.

**3.2 Compose `PageShell`, not `ChapterShell`.** Per `.claude/plans/shared/responsive-breakpoints.md` §6 Step 11, `PageShell` already provides TopNav + `<main id="main">` + Footer with the correct content max-width. The 404 page drops into that shell unmodified. "ErrorLayout" in the spec is NOT a new layout component — it is a visual variant of `PageShell` achieved by the page body (the centered hero + absence of prose wrapper). Decision: do NOT create a separate `ErrorLayout.astro`.

**3.3 One new hero component (`ErrorHero`).** The "4 0 4" display + headline + intro copy is a named, testable unit that might be reused for future error states (410 Gone, 500 Service Unavailable in a Phase 2 dedicated error catcher). Flagged for component-library addition.

**3.4 Search reuse.** The spec asks for a prominent search box. We already commit to one `SearchModal` globally (per `.claude/plans/shared/responsive-breakpoints.md` Step 8 + `.claude/plans/shared/accessibility.md` keybindings). Rather than inline a second Pagefind UI mount on 404, we render a static `<button>` that looks like an input ("Search for something…") and opens the same `SearchModal` via `document.getElementById('search-modal')?.showModal()`. Autofocus on page load is applied to that button, so keyboard users can press Enter to open search immediately.

**3.5 `noindex, follow` is load-bearing.** Without it, Google may index the 404 HTML body as a regular page. The `<SEO>` component takes `noindex: true` and emits the correct `<meta name="robots">`. Canonical is self-referencing (`/404`) rather than pointing to home, because Google treats a 404 with a canonical to the homepage as a soft 404 signal.

**3.6 Announce recovery options to SR users.** On page load, the view-transition-a11y handler (`.claude/plans/shared/accessibility.md` Step 5) focuses the H1 and announces the `<title>`. That same flow covers 404 — SR users hear "Page Not Found — GyanDev" and land on "This page wandered off" as the first focused heading. Search shortcut `/` and `⌘K` still work via `keybindings.ts`.

## 6. Implementation Steps (Ordered)

1. **Confirm `src/pages/404.astro` stub exists** from `.claude/plans/shared/routing-and-urls.md` Step 6. Replace the stub body with the real page.

2. **Create `src/components/pages/404/ErrorHero.astro`:**
   - Renders a visually-hidden H1 "This page wandered off" (for SR + focus target from view-transition-a11y) AND a large visual "4 0 4" display with `aria-hidden="true"`.
   - Below: two lines of intro copy ("We couldn't find what you were looking for." / "Try one of these:").
   - No props.
   - Uses CSS Grid with `place-items: center; min-height: 60dvh;` so it is vertically centered within `<main>`.
   - Display text uses `font-family: var(--font-serif)` + `font-size: clamp(4rem, 12vw, 8rem)`; on mobile caps at ~80px per spec §5 Responsive.
   - Done when: `<h1>` is focusable via `view-transition-a11y.ts` handler, VoiceOver reads "This page wandered off", and the "404" glyphs are announced as nothing.

3. **Create `src/components/pages/404/RecoveryLinks.astro`:**
   - Static markup only — no data fetching.
   - Renders a `<button type="button">` styled like a search input (label "Search for something…") with `data-opens="search-modal"`.
     - Inline script: on click, call `document.getElementById('search-modal')?.showModal()`.
     - `autofocus` attribute so first Tab lands here (actually — use explicit script: `requestAnimationFrame(() => el.focus())` inside `astro:page-load` so ClientRouter navigations also autofocus). Gate behind `prefers-reduced-motion` check: always focus regardless, but skip any entrance animation.
   - Renders an `<ol aria-label="Go to">` with three items: Home, All Courses, featured course. The featured-course label is resolved at build time from the same `src/content/featured.yml` used by the home page (see `.claude/plans/pages/home.md` §6) — if the file doesn't exist yet, hard-code "Node.js" + URL `/courses/nodejs`.
   - No props in Phase 1. In Phase 2 accept a `featuredCourse` prop if home and 404 diverge.
   - Done when: Tab order is Search button → Home → All Courses → Featured; each link is a real `<a href>`.

4. **Create `src/components/pages/404/ReportLink.astro`:**
   - Renders: `Broken link? <a href="https://github.com/sonushahuji4/gyandev/issues/new?labels=broken-link&template=broken-link.md">Let us know on GitHub.</a>`.
   - The GitHub issue template URL is a guess — verify at PR time. If the template does not exist, fall back to the bare new-issue URL.
   - `rel="noopener noreferrer"` because `target` is implicit internal (actually — no target, opens in same tab, so `rel` not strictly required; keep for defense).
   - Done when: link works, has visible focus ring, no JS.

5. **Replace `src/pages/404.astro` body:**
   ```astro
   ---
   import PageShell from '../components/layout/PageShell.astro';
   import ErrorHero from '../components/pages/404/ErrorHero.astro';
   import RecoveryLinks from '../components/pages/404/RecoveryLinks.astro';
   import ReportLink from '../components/pages/404/ReportLink.astro';
   import { canonicalFor } from '../lib/routes';

   const canonical = canonicalFor('/404');
   ---
   <PageShell
     title="Page Not Found"
     description="The page you're looking for doesn't exist. Search GyanDev or browse courses."
     canonical={canonical}
     noindex={true}
   >
     <ErrorHero />
     <RecoveryLinks />
     <ReportLink />
   </PageShell>
   ```
   Note: `PageShell` forwards `title`, `description`, `canonical`, `noindex` to `BaseLayout` → `<SEO>` — no direct SEO import here.
   - Done when: `npm run build` emits `dist/404.html` (not `dist/404/index.html`), View Source shows `<meta name="robots" content="noindex, follow">` and `<link rel="canonical" href="https://gyandev.org/404">`.

6. **Smoke-test the 404 HTTP behavior in CI.** Add one line to `scripts/validate-slugs.mjs` (routing plan) OR a new `scripts/check-404.mjs` run inside CI: after the build, `curl -s -o /dev/null -w "%{http_code}" http://localhost:4321/definitely-not-a-page` (dev server) must return `404`. Also assert `dist/404.html` exists and is < 30 KB uncompressed.
   - Done when: CI fails if someone accidentally changes `404.astro` to return 200 or blows the size budget.

7. **Verify keybindings remain active on 404.** No code change — just a manual check. `/` and `⌘K` must still open the global `SearchModal`. If they don't, `keybindings.ts` in `BaseLayout` is misconfigured — escalate to the a11y plan.

## 8. Code Patterns

**Pattern: Self-canonical on intentionally non-indexed pages.** Don't canonical a 404 to "/" — Google treats that as a soft-404 signal. Always self-canonical:
```astro
<PageShell canonical={canonicalFor('/404')} noindex={true} ...>
```

**Pattern: Focus the real H1 for SR, hide the visual "404" glyphs.**
```astro
<h1 class="sr-only">This page wandered off</h1>
<p aria-hidden="true" class="hero-glyphs">4 0 4</p>
<p class="headline" aria-hidden="true">This page wandered off</p>
```
Visual readers see the big headline; SR users hear it via the H1; "404" is announced as nothing.

**Pattern: Search trigger is a button, not an input.** A real `<input>` on 404 would mean wiring Pagefind twice (once here, once in the global modal). Instead, make the "input" a `<button>` that opens the shared modal:
```astro
<button type="button" class="search-trigger" aria-haspopup="dialog">
  <SearchIcon aria-hidden="true" />
  <span>Search for something…</span>
</button>
<script>
  document.querySelector('.search-trigger')
    ?.addEventListener('click', () => document.getElementById('search-modal')?.showModal());
</script>
```

**Pattern: Zero JS beyond the inline search trigger.** No `client:*` islands. No dynamic imports. The page must render and respond without any bundle.

## 9. Testing Strategy

**Build:**
- `npm run build` produces `dist/404.html`.
- File size < 30 KB uncompressed.
- View Source contains the expected `<meta>` + `<link rel="canonical">`.

**HTTP behavior:**
- Deploy preview: `curl -I https://<preview>.pages.dev/definitely-not-a-page` returns `HTTP/2 404`.
- Same URL returns the rendered 404 body (not Cloudflare's generic "This site is not configured" page).

**A11y:**
- axe-core: zero serious/critical on `/404`.
- VoiceOver walk: announces "Page Not Found — GyanDev", lands on "This page wandered off" as H1, does NOT read "four oh four" anywhere.
- Keyboard: first Tab hits SkipLink → second Tab hits search trigger → third/fourth/fifth hit the three recovery links → then ReportLink → then TopNav items.
- Focus ring visible on every focusable element per `global.css` `:focus-visible` rule.

**Perf:**
- Lighthouse on `/404`: Performance ≥ 90, a11y ≥ 95.
- DevTools Network: zero JS requests beyond the inline search trigger.

**Manual:**
- Mobile 375px: "404" glyphs don't overflow, CTAs stacked readable.
- Desktop 1920px: glyphs capped at ~128px, page doesn't feel empty (generous vertical centering via `place-items: center`).

**Regression:**
- If someone adds a new top-level route whose name collides with `404`, `validate-slugs.mjs` (routing plan) flags it. We reserve `404` in the RESERVED_SLUGS set.

## 11. Risks and Mitigations

- **Risk: Cloudflare Pages serves Cloudflare's default 404 instead of our `404.html`.**
  - Likelihood: low (serving `404.html` is default Pages behavior)
  - Impact: high (branding / UX regression)
  - Mitigation: explicit CI check in Step 6 hits a non-existent path on the preview deployment. If it returns the Cloudflare generic page, the file didn't build or Pages config is wrong.

- **Risk: SEO accidentally indexes the 404 body because `noindex` got dropped.**
  - Likelihood: low (type-safe BaseLayout prop)
  - Impact: medium (Search Console flags, traffic confusion)
  - Mitigation: `scripts/validate-schema.mjs` (seo plan) can be extended to assert `/404` emits `noindex` — add a one-line assertion.

- **Risk: Someone canonicalizes `/404` to `/` in a "cleanup" PR.**
  - Likelihood: medium
  - Impact: high (soft-404 signal to Google)
  - Mitigation: leave a one-line comment in `404.astro`: `// Self-canonical is deliberate — do not redirect to '/'.` Include in PR template review checklist.

- **Risk: The search trigger autofocus steals focus from SkipLink.**
  - Likelihood: medium
  - Impact: medium (a11y regression)
  - Mitigation: `view-transition-a11y.ts` focuses `<h1>` first; the search trigger uses `autofocus` only as a fallback and the manual `requestAnimationFrame` focus runs AFTER the a11y handler. Tab order verified in Step 7 manual check.

- **Risk: A future reusable `ErrorLayout.astro` is built anyway by a well-meaning contributor.**
  - Likelihood: medium
  - Impact: low (duplicate layout component)
  - Mitigation: document in §3.2 of this plan — `PageShell` is the layout; only the hero component is new. If a second error page surfaces (410/500), reconsider.
