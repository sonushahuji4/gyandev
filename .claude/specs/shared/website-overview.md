# Website Overview — Index

This is the **map** of GyanDev's website specifications. It does not contain details — each topic has its own dedicated spec file. Use this document to find the right spec to read or update.

## How This Folder Works

- `specs/shared/` — applies to every page on the site
- `specs/pages/` — one file per page type
- `specs/courses/` — per-course specifications
- Every spec in this folder has a matching **plan** in `.claude/plans/` at the same path

## Key Decisions (Locked)

These decisions affect every spec in this folder. Do not reopen without PM approval.

1. **Seasons are metadata, not URLs** — `/courses/nodejs/origin-story`, not `/courses/nodejs/season-1/origin-story`
2. **Three-tab chapter model** — Full Notes, Quick Revision, Flow Diagram; Full Notes is canonical
3. **No user accounts** — everything anonymous, localStorage for progress
4. **Static-first** — no server-side rendering at runtime, no database
5. **Phase 1 = 10 pages** — see page inventory above

## When to Read What

- **Starting a new page?** → Read `shared/*.md` first, then the page's own spec
- **Changing URL structure?** → Update `shared/routing-and-urls.md` only
- **Adding accessibility requirements?** → Update `shared/accessibility.md` only
- **Building a new MDX component?** → Update `plans/shared/component-library.md`
- **Adding a new course?** → Create `specs/courses/[name]/` and follow existing courses as template
