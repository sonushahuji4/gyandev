# 404 Not Found Page

The fallback page for any URL that doesn't match a real page. Critical for UX — broken links must lead somewhere helpful, not to a dead end.

## 1. Purpose

1. **Acknowledge** the user hit a wrong URL
2. **Redirect** them to useful content quickly
3. **Help them find** what they actually wanted (search)
4. **Preserve their trust** (friendly tone, not blame)

## 2. Target Audience

- Users hitting outdated links (slugs that changed)
- Users who mistyped a URL
- Referrers with broken links from other sites
- Search crawlers finding removed content

### Sections

**Hero**:
- Large "404" display text (Instrument Serif, 120px+)
- Friendly message ("This page wandered off", "Not found", etc.)
- 2-line explanation

**Search box**:
- Prominent, full-width
- Placeholder: "Search for something..."
- Opens Pagefind search on submit
- Autofocus after page load

**Quick links**:
- Home
- All Courses
- Most popular course (currently Node.js)
- (Phase 2) "Popular chapters" based on analytics

**Report**:
- "Broken link? Let us know on GitHub."
- Link to GitHub issue with prefilled "Broken link" template

## 4. Layout

Uses ErrorLayout — minimal:
- No sidebars
- No TOC
- Centered content
- Single column

## 5. Responsive Behavior

- Mobile: "404" smaller (80px), tighter spacing
- Desktop: "404" large, generous spacing
- Search input: always full width on mobile, constrained on desktop

## 6. Interactions

- Search: typing triggers Pagefind autocomplete
- Keyboard: Tab moves through links, Enter activates
- Focus: search input autofocused on load

## 7. HTTP Behavior

- **Status code**: 404 (not 200)
- Cloudflare Pages serves `/404.html` as the fallback for any unmatched route
- No redirect (404 is the final state, not a redirect)

## 8. Meta Tags

```html
<title>Page Not Found — GyanDev</title>
<meta name="description" content="The page you're looking for doesn't exist. Search GyanDev or browse courses.">
<meta name="robots" content="noindex, follow">
<link rel="canonical" href="https://gyandev.org/404">
```

## 9. Structured Data

None — 404 pages don't need schema.

## 10. Performance

- Minimal page weight (< 30KB)
- No JS required (search can use Pagefind loaded on demand)
- Loads instantly from Cloudflare edge

## 11. Accessibility

- H1: "This page wandered off" (not "404" — screen readers say "four oh four" oddly)
- Search input labeled "Search GyanDev"
- Links have descriptive text (not "click here")
